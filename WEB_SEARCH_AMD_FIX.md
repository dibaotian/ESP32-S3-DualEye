# 修复：搜索 AMD 股价失败

## 问题描述

**用户报告**："为什么搜索不到 AMD 的股价？"

**日志证据**：
```
2026-06-28 10:42:45 - STT: '查询一下最近AMD的股价。'
2026-06-28 10:42:48 - answer: '抱歉，我暂时查不到 AMD 的股价，你可以打开股票软件或者搜索引擎看看...'
```

**问题**：
- ❌ vLLM **没有调用** web_search 工具
- ❌ 直接返回了拒绝的回答
- ✅ 网络搜索功能正常（其他查询如 NBA、世界杯都能调用）

## 根本原因

### 工具描述不够明确

**原描述**（codex_vllm_proxy.py:542）：
```python
"description": "搜索网络信息。用户要求搜索、查找、查询实时信息时调用。"
```

**问题分析**：
1. **"搜索网络信息"太模糊**
   - 用户说"查询AMD股价"
   - 模型可能认为这是"股票查询"而不是"网络搜索"
   - 没有明确列出适用场景

2. **缺少关键词提示**
   - 没有提到：股价、新闻、赛程等具体应用
   - 模型不知道这些场景应该用 web_search

3. **与实际能力不匹配**
   - 工具描述说"实时信息"
   - 但没说包括"股价、汇率、价格"等

## 解决方案

### 改进工具描述

**修复前**：
```python
{
    "name": "web_search",
    "description": "搜索网络信息。用户要求搜索、查找、查询实时信息时调用。",
    "parameters": {
        "type": "object",
        "properties": {
            "query": {
                "type": "string",
                "description": "搜索关键词"
            }
        },
        "required": ["query"]
    }
}
```

**修复后**：
```python
{
    "name": "web_search",
    "description": "搜索网络信息。用于查询实时信息，包括：新闻、赛程、股价、汇率、产品价格、公司信息等。用户说'搜索'、'查询'、'找一下'时调用。",
    "parameters": {
        "type": "object",
        "properties": {
            "query": {
                "type": "string",
                "description": "搜索关键词，例如：'AMD股价'、'NBA赛程'、'ChatGPT是什么'"
            }
        },
        "required": ["query"]
    }
}
```

### 改进要点

1. ✅ **明确列举适用场景**
   - 新闻、赛程、**股价**、汇率、产品价格、公司信息
   - 让模型知道"股价"也属于搜索范畴

2. ✅ **提供具体示例**
   - query 参数描述中加入示例：'AMD股价'、'NBA赛程'
   - 让模型学习如何构造查询

3. ✅ **明确触发词**
   - 说明"搜索"、"查询"、"找一下"时调用
   - 降低模型的判断难度

## 工具描述设计原则

### 好的工具描述应该包含：

1. **功能说明** - 这个工具做什么
   ```
   "搜索网络信息"
   ```

2. **适用场景** - 什么时候用（明确列举）
   ```
   "用于查询实时信息，包括：新闻、赛程、股价、汇率、产品价格、公司信息等"
   ```

3. **触发条件** - 什么情况下调用
   ```
   "用户说'搜索'、'查询'、'找一下'时调用"
   ```

4. **参数示例** - 如何构造参数
   ```
   "搜索关键词，例如：'AMD股价'、'NBA赛程'、'ChatGPT是什么'"
   ```

### ❌ 避免模糊描述

**不好的示例**：
```python
"description": "搜索工具"  # 太简单
"description": "查询信息"  # 太模糊
"description": "用于搜索"  # 没说搜索什么
```

**好的示例**：
```python
"description": "搜索网络信息。用于查询实时信息，包括：新闻、赛程、股价、汇率、产品价格、公司信息等。用户说'搜索'、'查询'、'找一下'时调用。"
```

## 相关问题

### Q1: 为什么 NBA 赛程能搜索，AMD 股价不能？

**回答**：
- NBA 赛程："赛程"明显需要实时查询，工具描述隐含支持
- AMD 股价："股价"没在工具描述中提及，模型不知道该用这个工具

### Q2: 是不是模型能力问题？

**回答**：
- 不是模型能力问题
- 是**工具描述不够明确**的问题
- 模型只能根据工具描述判断何时调用
- 描述越详细，调用越准确

### Q3: 为什么不直接说"用于所有查询"？

**回答**：
- 太宽泛会导致误用
- 例如："你叫什么名字"也是"查询"，但不应该搜索网络
- 需要**明确边界**：哪些查询需要搜索，哪些不需要

### Q4: 如何测试工具描述是否足够好？

**方法**：
1. 列出典型场景：新闻、赛程、股价、天气、价格...
2. 逐个测试：用户怎么问，工具会不会被调用
3. 失败的案例：在描述中明确列举
